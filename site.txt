    <!-- Why Section -->
    <section class="why">
        <h2>Why This Exists</h2>
        <p>
            Setting up native package repositories is tedious, which is why many developers 
            on small hobby projects don't even bother. I can't tell you how many times I've found 
            a bespoke application on the web and installed it manually only to completely forget about it. 
            Mainly I just wanted to see how difficult it would be to create a solution to handle package 
            distribution automatically, as just a user of the software and not the maintainer. 
            Thankfully, in Python this was pretty easy to get working. 
            I tried doing this in JavaScript but I almost had an aneurism trying to get the necessary 
            decryption libraries working correctly.
        </p>
    </section>

    <!-- Installation -->
    <section class="installation">
        <h2>Quick Start</h2>
        <pre><code># Install globally
npm install -g no-package-left-behind

# Generate APT repo from GitHub repository
nplb generate owner/repo --output ./apt-repo</code></pre>
    </section>

    <!-- Features -->
    <section class="features">
        <h2>Key Features</h2>
        <ul>
            <li>Single command repository generation</li>
            <li>Automatic package signing and verification</li>
            <li>Compatible with all major Debian-based distributions</li>
            <li>Zero configuration required</li>
        </ul>
