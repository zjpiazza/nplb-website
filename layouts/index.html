Generate APT repositories from GitHub repos with a simple bash script. No complexity, no hassle - just packages ready to distribute.

Why This Exists

Setting up native package repositories is tedious, which is why many developers on small hobby projects don't even bother. I can't tell you how many times I've found a bespoke application on the web and installed it manually only to completely forget about it. Mainly I just wanted to see how difficult it would be to create a solution to handle package distribution automatically, as just a user of the software and not the maintainer. Thankfully, in Python this was pretty easy to get working. I tried doing this in JavaScript but I almost had an aneurism trying to get the necessary decryption libraries working correctly.

Quick Start

# Install globally
npm install -g no-package-left-behind

# Generate APT repo from GitHub repository
nplb generate owner/repo --output ./apt-repo
Disclaimer

This is beta software meant mainly as a proof of concept. The topic of Software Supply Chain security is incredibly complex and the current implementation of this project has many potential issues from a security perspective. There is a single PGP key used to sign all packages. Depending on how far I end up taking this project, I might provide more sophisticated signing capabilities, but I just want to let any potential users of this service be aware of the potential risks.

The API for this project does not have any test coverage yet, so it is quite likely users could encounter some bugs. If you find this service useful and encounter an issue, I welcome anyone to submit an issue here: New Issue

Key Features

  • Single command repository generation
  • Automatic package signing and verification
  • Compatible with all major Debian-based distributions
  • Zero configuration required
